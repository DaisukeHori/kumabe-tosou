## 2. shop シミュレータ操作チェック (`/shop`)

- [ ] サイズ・個数・グレードを変更すると概算金額が変わる
- [ ] 極端な値 (最小/最大) を入れても表示が壊れない
- [ ] シミュレータ結果から `/contact` への導線が動作する

## 3. contact フォーム動作チェック (`/contact`)

- [ ] 空のまま送信ボタンを押すと、各必須項目 (お名前・メールアドレス・お問い合わせ種別・内容・同意チェック) に validation error が表示される
- [ ] 必須項目が未入力の状態では実際には送信されない (画面が「受付完了」に切り替わらない)
- [ ] メールアドレスに不正な形式 (例: `abc`) を入れるとエラーが出る
- [ ] 内容欄に10文字未満を入れるとエラーが出る (「10文字以上でご記入ください」)
- [ ] 内容欄に2000文字を超える文章を入れるとエラーが出る
- [ ] プライバシーポリシーの同意チェックを外したまま送信するとエラーが出る
- [ ] チェック内の「プライバシーポリシー」リンクをクリックすると `/privacy` に遷移する
- [ ] 全項目を正しく入力して送信すると「お問い合わせを受け付けました(モック)」の画面に切り替わる
- [ ] 上記完了画面に「このフォームはモック段階のため、実際の送信・保存は行われていません」の注記がある
- [ ] 「もう一度入力する」ボタンで入力画面に戻れる
- [ ] ブラウザの開発者コンソールに送信内容がログ出力される (外部送信は行われない設計の確認、任意)

## 4. 共通ヘッダー・フッター・SEO チェック

- [ ] 全ページでヘッダーのナビゲーションから他ページへ遷移できる
- [ ] 全ページでフッターの `特定商取引法に基づく表記` / `プライバシーポリシー` リンクが機能する
- [ ] ブラウザタブのタイトルがページごとに変わる (例: `会社案内 | 隈部塗装`)
- [ ] SNS 共有プレビュー (OGP) で各ページのタイトル・画像が正しく出る (Facebook Sharing Debugger 等で任意確認)
- [ ] `/tokushoho` ページは検索結果に出さない設定 (noindex) のため、Google 等でインデックスされていないことを後日確認 (公開後でOK)

## 5. 既知の制約 (Phase 0.5 時点)

- contact フォームは実送信されないモックです (Phase 1 で Supabase 接続予定)
- works (施工事例) / voices (お客様の声) はダミーコンテンツです (実データは Phase 1 以降で差し替え)
- 実ブラウザでの E2E 検証 (Chrome 等での実機操作) は本チェックリストで堀さんが実施してください
